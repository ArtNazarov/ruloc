# RULOC - Сетевой монитор безопасности

RULOC — это инструмент сетевого мониторинга, который отслеживает активные TCP-соединения и автоматически реагирует на соединения из стран, отличных от России. Доступны две версии: инструмент командной строки (CLI) и графический инструмент (GUI) с визуализацией в реальном времени в виде радара.

## Скриншот

[ruloc GUI](https://dl.dropbox.com/scl/fi/fyyswrd2meloomerrtue5/ruloc-gui-variant.png?rlkey=gzhkr6k60tts65dr14vdiirqr&st=ilnlq8g5)

## Возможности

- Мониторинг активных TCP-соединений в реальном времени
- Определение страны по IP-адресу (используется база GeoLite2)
- Автоматическая блокировка IP через `ufw`
- Завершение процессов с нежелательными соединениями
- Только соединения из `RU` считаются разрешёнными

### Дополнительные возможности GUI версии:
- **Визуализация в виде радара** с вращающейся линией сканирования
- **Лучи стран** под фиксированными углами для лёгкой идентификации
- **Точки соединений** в реальном времени на радаре
- **Пульсирующий след сканирования** с вращением по часовой стрелке (1 оборот за 3 секунды)
- **Цветовая индикация угроз** (зелёный для RU, красный для высокого риска, жёлтый для среднего)
- **Панель статистики** в реальном времени с количеством соединений
- **Журнал событий** с сообщениями и временными метками
- **Интерактивные элементы управления** (Запуск/Остановка мониторинга, Очистка журнала)

## Требования

- Python 3.6+
- Linux с `ufw` (для блокировки)
- Права `sudo` (для добавления правил `ufw`)
- Файл базы данных `GeoLite2-Country.mmdb` в директории со скриптом

## Установка и настройка

### 1. Клонирование репозитория

```bash
git clone <your-repo-url>
cd <project-directory>
```

### 2. Настройка виртуального окружения

```bash
# Создание виртуального окружения
python3 -m venv venv

# Активация виртуального окружения
source venv/bin/activate   # Для Linux/macOS
# или
venv\Scripts\activate      # Для Windows (при локальном использовании)
```

### 3. Установка зависимостей

```bash
# Для CLI версии
pip install psutil geoip2

# Для GUI версии (включает matplotlib для визуализации радара)
pip install psutil geoip2 matplotlib
```

### 4. Загрузка базы данных GeoIP

Скачайте бесплатную базу `GeoLite2-Country.mmdb` с [официального сайта MaxMind](https://dev.maxmind.com/geoip/geolite2-free-geolocation-data) и поместите её в директорию со скриптом.

```bash
wget https://github.com/P3TERX/GeoLite2.mmdb/raw/master/GeoLite2-Country.mmdb
```

### 5. Настройка прав sudo

Для работы блокировки через `ufw` скрипту требуются права `sudo` без запроса пароля (рекомендуется только для конкретной команды):

```bash
sudo visudo -f /etc/sudoers.d/ufw-ruloc
```

Добавьте строку:
```
your_username ALL=(ALL) NOPASSWD: /usr/sbin/ufw deny from *
```

## Использование

### CLI версия (ruloc.py)

#### Базовый синтаксис

```bash
python3 ruloc.py [--ufw-block-detected] [--close-proc]
```

#### Примеры использования

**Только блокировка IP через UFW:**
```bash
python3 ruloc.py --ufw-block-detected
```

**Только завершение процессов:**
```bash
python3 ruloc.py --close-proc
```

**Блокировка и завершение процессов:**
```bash
python3 ruloc.py --ufw-block-detected --close-proc
```

**Запуск в фоновом режиме:**
```bash
nohup python3 ruloc.py --ufw-block-detected --close-proc &
```

### GUI версия (ruloc-gui-variant.py)

#### Базовый синтаксис

```bash
python3 ruloc-gui-variant.py [--ufw-block-detected] [--close-proc]
```

#### Примеры использования

**Только мониторинг (без автоматических действий):**
```bash
python3 ruloc-gui-variant.py
```

**Мониторинг и блокировка IP через UFW:**
```bash
python3 ruloc-gui-variant.py --ufw-block-detected
```

**Мониторинг и завершение процессов:**
```bash
python3 ruloc-gui-variant.py --close-proc
```

**Полная защита (блокировка + завершение):**
```bash
python3 ruloc-gui-variant.py --ufw-block-detected --close-proc
```

#### Обзор GUI интерфейса

При запуске окно GUI отображает:

1. **Левая панель - Монитор радара:**
   - Вращающийся радар сканер (по часовой стрелке, 1 оборот за 3 секунды)
   - Цветные лучи для разных стран под фиксированными углами:
     - 0° (Вверх): Россия (RU) - Зелёный
     - 30°: США - Красный
     - 60°: Китай - Красный
     - 90°: Германия - Жёлтый
     - 120°: Великобритания - Жёлтый
     - 150°: Франция - Жёлтый
     - 180°: Япония, Неизвестные страны - Жёлтый/Красный
     - 200°-340°: Другие страны
   - Цветные точки, представляющие активные соединения
   - Новые соединения появляются ближе к центру
   - Соединения исчезают через 30 секунд

2. **Правая панель:**
   - **Управление:** Кнопки Остановка/Запуск мониторинга, Очистить журнал
   - **Статистика:** Количество активных/RU/не-RU/неизвестных соединений, заблокированные IP
   - **Активные соединения:** Таблица с IP, Страной, Процессом, PID, Временем
   - **Журнал событий:** Журнал всех обнаруженных событий с временными метками

3. **Строка состояния:**
   - Показывает текущее количество соединений и статус сканирования

#### Горячие клавиши

- `Ctrl+C` в терминале (CLI версия) - Остановка мониторинга
- Закрыть окно (GUI версия) - Выход из приложения

## Как это работает

### CLI версия
1. Скрипт сканирует все установленные TCP-соединения каждые 3 секунды
2. Для каждого внешнего IP определяется страна по базе GeoIP
3. Если страна не `RU`:
   - Выводится сообщение в консоль
   - При `--ufw-block-detected` добавляется правило `ufw deny from <IP>`
   - При `--close-proc` завершается процесс, владеющий соединением

### GUI версия
1. Та же логика мониторинга, что и в CLI версии
2. Визуализация соединений на полярном графике радара:
   - Угол определяется страной (Россия на 0°, неизвестные не-RU на 180°)
   - Радиус определяется возрастом соединения (новые = ближе к центру)
   - Цвет точки указывает на уровень риска
3. Анимированная линия сканирования вращается по часовой стрелке со скоростью 120° в секунду
4. Пульсирующий след показывает недавний путь сканирования
5. Обновление таблицы соединений и статистики в реальном времени

## Уровни риска по странам

- **🟢 Зелёный (Россия - RU):** Разрешённые соединения
- **🔴 Красный (Высокий риск):** США, Китай и другие страны с высоким уровнем риска
- **🟡 Жёлтый (Средний риск):** Германия, Великобритания, Франция, Япония и др.
- **🔴 Красный (Неизвестный):** Неизвестные страны обрабатываются как не-RU и появляются на 180°

## Важные замечания

- Скрипт не различает входящие и исходящие соединения — проверяются **все** установленные соединения
- Блокировка через UFW перманентна до ручного удаления правила
- Завершение процессов может быть опасным — используйте опцию `--close-proc` с осторожностью на production-серверах
- Для корректной работы необходима актуальная база GeoIP
- GUI версия требует X11 дисплей сервер (работает в средах рабочего стола Linux)
- Визуализация радара показывает соединения в течение 30 секунд после их закрытия

## Устранение неполадок

### GUI не запускается или показывает ошибку импорта
Убедитесь, что matplotlib установлен:
```bash
pip install matplotlib
```

### Ошибка значения alpha в GUI
Это было исправлено в последней версии. Убедитесь, что вы используете обновлённый скрипт с правильным ограничением alpha.

### База данных GeoIP не найдена
Скачайте файл базы данных и поместите его в ту же директорию, что и скрипт:
```bash
wget https://github.com/P3TERX/GeoLite2.mmdb/raw/master/GeoLite2-Country.mmdb
```

### Отказано в доступе для UFW
Убедитесь, что вы настроили права sudo правильно, или запустите с sudo:
```bash
sudo python3 ruloc.py --ufw-block-detected
```

### Нет соединений на радаре
Радар показывает только текущие установленные соединения. Выполните какую-либо сетевую активность (просмотр веб-страниц, использование curl и т.д.), чтобы увидеть появление соединений.

## Вопросы производительности

- **CLI версия:** Минимальное воздействие на систему (~5-10% CPU на одном ядре)
- **GUI версия:** Умеренное воздействие на систему (~10-20% CPU из-за анимации)
- Обе версии используют ~50-100 MB ОЗУ в зависимости от количества соединений
